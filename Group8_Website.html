Project Overview

Why Amazon Needs a Recommendation Engine

Amazon sells 350+ million products. Without personalisation, customers face choice overload and conversion drops to 2–3%. Our three models solve this at different stages of the user journey.

🏆

Popularity-Based

Bayesian weighted scoring ranks the best products globally. Same top-5 for every user — solves cold-start for brand new users with zero history.

Model 1 — Cold-Start
🔗

Item-Based CF

Cosine similarity between product rating vectors. Finds products most similar to what the user already liked. Amazon's patented approach since 2001.

Model 2 — Production
👥

User-Based CF

Finds the 10 most similar users and recommends products they loved that the target user hasn't seen. More serendipitous — discovers unexpected gems.

Model 3 — Discovery
📊

J-Curve Bias

60.2% of reviews are 5 stars. Our Bayesian score corrects for this — the same system IMDB uses for movie rankings to prevent 1-review products from ranking first.

Data Insight
🧮

Sparsity Challenge

99.97% of the user-product matrix is empty. Only 20,000 ratings out of 60 million possible pairs. We filter to ≥3 ratings to ensure enough signal.

Core Challenge
🚀

Recommendation Models

Three Approaches, One Goal

Each model solves a different part of the recommendation problem. Together they cover every user scenario.

1
Popularity-Based Cold-Start
Same top-5 for every user — no rating history required

Recommends the globally best products using a Bayesian weighted score. A product with 1 five-star review is NOT ranked above one with 500 reviews at 4.8★. When n (reviews) is small, the score is pulled toward the global mean C — protecting against fake reviews and noise.

✓ Completely solves cold-start — no user history needed
✓ Protects against fake review manipulation
✓ Same system used by IMDB for movie rankings
✗ No personalisation — same for everyone
Bayesian Weighted Score
score = (n / (n+m)) × avg_rating
      + (m / (n+m)) × C

// n = reviews for this product
// m = 5 (minimum threshold)
// C = 4.19 (global mean rating)

When n → 0, score → C. When n → ∞, score → avg_rating. Shrinkage toward the mean is the key insight.

2
Item-Based CF Recommended
Amazon's patented approach — products rated alike are similar

Computes cosine similarity between every pair of product rating vectors. For a user, scores each unrated product by its average similarity to already-rated ones. Pre-computed offline — instant at serving time. Amazon patented this in 2001 and still uses it today.

✓ Personalised recommendations
✓ Pre-computable offline — millisecond serving
✓ Item similarities stable over time
✓ Scales to millions of products
✗ Needs rating history (cold-start issue)
Cosine Similarity
sim(X,Y) = (X·Y) / (‖X‖ × ‖Y‖)

// X, Y = product rating vectors
// 0 = completely different
// 1 = identical rating patterns
// Matrix: 2,274 × 2,274 product pairs

Each product is a vector of all ratings it received. Products rated similarly by the same users point in the same direction → high cosine similarity.

3
User-Based CF
Find your rating twins — recommend what they loved

Finds k=10 most similar users via cosine similarity, then recommends products they rated highly that the target user hasn't seen. The .drop(user_id) call is critical — without it the target user would always be their own neighbour (similarity = 1.0), giving useless circular recommendations.

✓ Highly personalised and serendipitous
✓ Discovers unexpected products
✗ O(n²) matrix grows with every new user
✗ Does not scale to millions of users
✗ Sensitive to sparse data
k-NN Neighbour Selection
neighbors = sim_df[user_id]
  .drop(user_id) # exclude self!
  .sort_values(ascending=False)
  .head(10).index

predicted = rating_matrix
  .loc[neighbors].mean(axis=0)
  .drop(already_rated)

.drop(user_id) before sorting — without this the target user is always their own best neighbour (sim=1.0).

Model Comparison

Trade-Off Analysis

No single model wins on every criterion. The right choice depends on the user context.

Criterion 🏆 Popularity 🔗 Item-Based CF ⭐ 👥 User-Based CF
Personalised?✗ No✓ Yes✓ Yes
Cold-Start✓ Handles it✗ Needs history✗ Needs history
Scalability✓ O(1) instant✓ Pre-computed✗ O(n²) per user
Sparsity Robust✓ Not affected⚠ Moderate✗ Low
Serendipity✗ Low⚠ Medium✓ High
Fake Review Resistant✓ Bayesian score——
Production Ready✓ Yes✓ Yes (Amazon)✗ Not at scale
Recommended Strategy: Popularity for new users (0–2 ratings) → Item-Based CF after 3+ ratings → User-Based CF for highly active users who want surprising discoveries.
